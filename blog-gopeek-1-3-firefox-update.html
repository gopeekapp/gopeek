What Is New in GoPeek 1.3 for Firefox

June 11, 2026 · 3 min read · By GoPeek Team

GoPeek 1.3 is now rolling out to Firefox users. If you have the extension installed, it updates automatically through the Firefox Add-ons store — no action needed. Here is what changed since 1.2.

Smarter Hover Detection

In 1.2, hovering a link while holding Shift immediately woke the background engine and fired a header bypass + network prefetch. If you were sweeping your mouse across a dense page, you could trigger a dozen background requests and cancel most of them a split-second later.

1.3 adds a 75ms pre-check. The extension waits to confirm your cursor has actually settled on the link before doing any heavy lifting. If you keep moving, nothing fires. The preview still appears at the same ~300ms mark when you stop, but your browser is no longer thrashing on every accidental hover.

Optional Long-Press Previews

1.3 introduces a second trigger method: hold-click any link to open a preview. No modifier key required. It is disabled by default, so the default experience is unchanged.

To enable it, open the GoPeek popup, toggle Long Press, and pick a duration between 300ms and 1000ms. If you drag the link more than 6 pixels, the timer cancels automatically — so it will not interfere with native link dragging or text selection.

Preloader Enabled by Default

The three-dot loading animation is now on by default. In 1.2 it was off, which meant some users saw a blank white flash while the iframe initialized. The preloader gives immediate visual feedback that the preview is loading. You can still disable it in settings if you prefer the raw snap.

Updated Settings Panel

The popup now includes the Long Press toggle and a Press Duration dropdown. The layout is otherwise identical to 1.2 — no settings were removed or renamed, so your existing preferences carry over unchanged.

Summary: 1.3 is leaner on background resources, gives you one more natural way to trigger previews, and removes the blank flash on first load. Everything else — drag, snap, bubble minimize, Multi-Peek, dynamic theme, and search selection — works exactly as it did in 1.2.
